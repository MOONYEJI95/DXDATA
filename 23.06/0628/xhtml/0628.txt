src 경로

- 속성
태그안에 속성이름="값" 의 형태로 작성해서 태그의 옵션을 설정하는 것
모든 태그에 사용할 수 있는 속성이 있고 태그별로 별도로 갖는 속성이 있음

공통속성
=>id:페이지 내에서 구분하기 위한 이름
=>class:묶음을 만들기 위한 이름
=>name:서버에게 데이터를 전달할 때 사용하기 위한 이름
=>data:태그 안에 데이터를 포함하고자 하는 경우

경로의 종류
1. 절대경로
: 변하지 않는 경로
: 웹의 URL이 절대경로(<a href="http://www.naver.com">네이버</네이버>
: c://?/?
: /var/???

2. 상대경로
: 현재 위치에서의 경로
./ 나 앞에 기호를 생략하면 현재위치(암기필요)
../: 상위 디렉토리(암기필요)

현재 작업 디렉토리를 git에 저장
- 디렉토리로 프롬프트를 이동해서 git init이라는 명령을 수행
.git 파일이 생성되고 이 파일이 생성되면 git이 이 소스 코드 버전 관리를 한다라는 의미가 됩니다.
git 명령이 수행되지 않는 경우 git이 설치된 디렉토리이 bin이라는 디렉토리를 path라는 환경 변수에 추가해주면 됩니다.

-관리할 파일을 등록
git add 파일명 파일명 ....
git add . : 현재 디렉토리에 모든 내용을 git이 관리하도록 설정
commit :현재 변경 내용을 로컬git(내컴퓨터)에반영
git commit -m "메세지..."
git commit -m "HTML을 처음 수행"

-변경 내용 확인 > git log

* 로컬 git 내용을 git hub 사이트에 업로드
- git hub 사이트에 repository를 생성하고 url을 복사
- 현재 git 디렉토리와 repository를 연결
- 코드를 업로드(push)
git remote add 이름(아무거나) https://github.com/MOONYEJI95/DXDATA.git
(git remote add origin https://github.com/MOONYEJI95/DXDATA.git)
git push 이름 브랜치이름
git push origin main   치고 로긴하면 됨

* 코드를 수정하고 수정한 내용을 git hub에 반영하려면
1. git add .
2. git commit -m "메세지"
3. git push 이름 브랜치이름(브랜치이름을 수정하면 별도의 장소에 보관)


실행하는 명령어 > bin에 있음
내pc>오른쪽버튼>설정>시스템 환경 변수 편집 > 환경변수 > path > 새로만들기 > 경로 붙여넣고 확인
